Accessing IPHAS images

Summary

This server provides access to the entire collection of pipeline-reduced images that have been obtained as part of the IPHAS survey of the Northern Galactic Plane. The data set consists of 268 180 CCD frames, recorded during 67 045 exposures across more than 300 nights between 2003 and 2012. The data volume is 1.6 TB.

All the images were obtained by the four-CCD Wide Field Camera (WFC) at the Isaac Newton Telescope (INT) in La Palma. The data were pipeline-processed by the Cambridge Astronomical Survey Unit (CASU). The FITS headers have been updated to include the new, globally re-calibrated DR2 photometric zeropoints however.

This page also provides a table containing the associated meta data and the quality control information that is required to select images. Note that the raw telescope data and the calibration frames are not available here, they may be found in the Isaac Newton Group Archive.

Downloading images

IPHAS images are uniquely identified by the combination of their run number, which identifies the exposure, and their CCD number, which identifies the detector within the camera (i.e. CCD number 1, 2, 3 or 4). Run and CCD numbers for a desired region or epoch may be looked up using the metadata table provided below. The run numbers may also be found in the INT's archive of observing logs.

When the run and CCD numbers of a desired frame are known, the URL of the image may be composed as follows:

http://www.iphas.org/data/images/r{first 3 digits of RUN}/r{RUN}-{CCD}.fits.fz

For example, the frame recorded by CCD #2 in run number 534002 is located at:

http://www.iphas.org/data/images/r534/r534002-2.fits.fz

Note that all the images are compressed using the Rice algorithm. Although this format is widely supported by astronomical software, the data can be decompressed using the funpack tool if necessary.

Warning

This data set includes all images, including those that were obtained under poor conditions (e.g. clouds, poor seeing, electronic noise). You should verify the quality of the images for your purpose, using the information given below.

Metadata table

Downloading metadata

A table detailing the run numbers, sky coordinates, observing dates, and quality information is available in FITS and ASCII format:

  • iphas-images.fits.gz (20 MB FITS);
  • iphas-images.txt (41 MB ASCII, limited set of columns).

The table summarises the quality of each image in the qcgrade column. The value of qcgrade is one of "A", "B", "C", or "D":

  • "A"-graded images were found to meet all our quality criteria;
  • "B"- and "C"-graded images do not meet the criteria in a strict sense, but may nevertheless be adequate for a range of scientific applications. For example, this category includes images that were obtained under sub-standard seeing (2 to 2.5 arcsec), but are fine otherwise.
  • "D"-graded images were excluded from DR2 over serious quality concerns and their use is discouraged.

Note that the quality grades were assigned on a field-by-field basis, during which the same-epoch exposures of the field in all three wavebands were evaluated at the same time. Hence, the D-graded category does contain some single-band exposures which are perfectly fine, but were graded poorly because of a quality problem in an accompanying exposure.

Some fields in the survey footprint were observed more than once, in an attempt to improve data quality. Column in_dr2 in the metadata table flags the best-available image for each field, and marks the images that were used to compile the DR2 source catalogue.

Column definitions

The columns in the metadata table are as follows:

Column Unit Description
run Number of the exposure, corresponding to the run number found in the observing logs of the Isaac Newton Telescope (INT).
ccd Number of the CCD of the INT Wide Field Camera (1, 2, 3, or 4).
url URL of the FITS file containing the image, e.g. "www.iphas.org/data/images/r534/r534002-2.fits.fz".
ra degrees Right Ascension (J2000) in the centre of the frame, i.e. at pixel position (x,y)=(1024,2048).
dec degrees Declination (J2000) in the centre of the frame, i.e. at pixel position (x,y)=(1024,2048).
band Waveband of the filter used. One of "r", "i", or "halpha".
utstart Universal Time (UTC) at the start of the exposure, given as a ISO 8601 timestamp (e.g. "2003-08-08T22:07:10.4").
fieldid Internal IPHAS identifier for the observation composed of the field number and the observing month, e.g. "0001o_aug2003".
qcgrade IPHAS DR2 quality score. One of "A++", "A+", "A", "B", "C" or "D". D-graded data is generally unsuited for scientific use due to e.g. clouds or electronic noise in the image.
in_dr2 Boolean column, true if the exposure passed all the quality control criteria and was selected for inclusion in IPHAS Data Release 2 (DR2). When this column is set to false, then it is likely that the frame suffers from severe quality problems (e.g. clouds).
exptime seconds Exposure time adopted in DR2.
seeing arcsec Average Full Width at Half Maximum (FWHM) of point sources in the CCD frame.
elliptic Average ellipticity of point sources in the CCD frame, defined as e = (1 - b/a) with b the semi-minor and a the semi-major axis.
skylevel counts/pixel Median sky brightness. An automatic 2D background-following algorithm is used to track and "remove" slowly varying background features such as image gradients etc.
skynoise counts/pixel Pixel noise at sky level, estimated using a robust MAD estimator for noise scaled to equivalent Gaussian rms value, ie. = MAD x 1.48, after removing large scale sky background variations. MAD = Median of the Absolute Deviations about the median.
photzp Photometric zeropoint, computed such that it absorbs the required corrections for atmospheric extinction, gain variations, exposure time, and the DR2 re-calibration shift (included only if the frame was part of DR2). Hence, the number counts stored in the frame can be converted into Vega-based magnitudes using: mag(Vega) = -2.5*log(pixel value) + PHOTZP.
confmap Filename of the confidence map, relative to their loctation on the IPHAS server: http://www.iphas.org/data/images/confmaps
ra_min degrees Right Ascension (J2000) in the western-most corner of the CCD frame.
ra_max degrees Right Ascension (J2000) in the eastern-most corner of the CCD frame.
dec_min degrees Declination (J2000) in the southern-most corner of the CCD frame.
dec_max degrees Declination (J2000) in the northern-most corner of the CCD frame.

Photometric calibration

Both the image headers and the metadata table provide a photometric zeropoint for each image (keyword PHOTZP). This zeropoint can be used to convert the pixel values (i.e. number counts) into Vega-based magnitudes as follows:

mag(Vega) = -2.5*log10(pixel value) + PHOTZP.

The PHOTZP value has been computed such that it absorbs the required corrections for atmospheric extinction, gain variations, exposure time, and the DR2 re-calibration shift. As these images still include moonlight and other sources of non-astronomical background, they can only support flux measurements that include a suitably-chosen local background subtraction.
